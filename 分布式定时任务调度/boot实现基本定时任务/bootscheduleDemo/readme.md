# springboot实现定时任务基本Demo
## 基于注解的静态定时任务
- 特点就是将cron表达式的周期时间直接以硬编码形式嵌入代码中，日后若有更改周期的需求，则只能通过重启应用项目达到目的。

## 基于接口的动态定时任务
- 解决静态任务定时的痛点：对于静态任务定时编码，现实情况往往不允许我们随便重启应用，于是便需要动态任务定时。
- 特点：将周期时间以软编码形式嵌入代码中

- 具体实现：
    1. 实现方式之一是利用数据库存储cron
    2. 另一种是利用广泛使用的Apollo配置中心，存储cron表达式
